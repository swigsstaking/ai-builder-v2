# Session 2026-04-07 (B) — Pages multiples

## Ce qui a été fait

### Phase 1 : Nouvelles sections (fondation)
- **Page.js** : ajouté `gallery`, `pricing`, `products`, `hours`, `menu` à l'enum
- **pageController.js** : ajouté 5 sections par défaut (hidden) + `SUBPAGE_DEFAULTS` avec des templates de sections spécifiques par type de sous-page (menu, gallery, pricing, team, hours, faq-page, services-detail)
- **10 templates** (5 frontend + 5 backend) : ajouté renderers pour les 5 nouvelles sections, stylisés par template
- **BlueprintPanel.jsx** : ajouté les 5 sections dans `SECTION_DEFS`
- **aiPageBuilder.js** : ajouté mapping AI + distribution d'images pour les nouveaux types

### Phase 2 : Migration multi-pages
- **vision.service.js** : enrichi le prompt OCR avec `pageMapping` (quelle page contient quel type de contenu) + `num_predict` 2048→3072
- **Migration.js** : ajouté `products`, `team`, `testimonials`, `openingHours`, `pageMapping` dans `extractedContent` + nouveau champ `pageStructure` (tableau de pages avec sections)
- **migration.service.js** : `normalizeAnalysis` enrichi
- **content-mapper.service.js** : nouvelle fonction `mapAnalysisToPages()` — crée homepage + sous-pages conditionnelles
- **migrationController.js** : `triggerMapping()` utilise `mapAnalysisToPages`, `createSiteFromMigration()` boucle sur `pageStructure`, `buildBasicPageStructure()` remplace l'ancien mapping
- **MigrationWizardPage.jsx** : UI multi-pages avec onglets par page
- **migrationStore.js** : ajouté `updatePageStructure`

### Phase 3 : Création de site multi-pages
- **ai.service.js** : nouvelle fonction `generateSubPageContent()` avec prompts dédiés par type de page
- **aiController.js** + **routes/ai.js** : endpoint `POST /ai/generate-subpage`
- **api.js** (frontend) : ajouté `generateSubPage`
- **SiteCreatePage.jsx** : presets par activité (`ACTIVITY_PAGE_PRESETS`), UI sélecteur de sous-pages, `handleCreate` multi-pages, passage de `subPageType` lors de la création

### Fix critique post-test
- **pageController.js** : ajouté `SUBPAGE_DEFAULTS` — sans ça, les sous-pages (menu, gallery, etc.) recevaient les 9 blocs universels au lieu des sections appropriées. Le `subPageType` dans la requête de création sélectionne maintenant les bonnes sections par défaut.

## Tests effectués (Chrome MCP)
- Création d'un restaurant "La Table du Valais" à Sion, style Élégant, palette Earth
- 5 pages créées avec succès : homepage, carte-menu, galerie, horaires, contact
- IA génère du contenu unique pour chaque page (~90s total pour 5 pages IA + contact + SEO)
- Build SSG réussi pour les 5 pages
- Rendu vérifié : menu avec catégories/plats/prix, horaires avec tableau jour par jour, galerie avec grille
