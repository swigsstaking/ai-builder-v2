### 4. Migration de site — Fix complet du pipeline OCR

#### Problème initial
La migration était complètement cassée : qwen3-vl timeout après 3+ minutes, fallback Claude Vision lent et payant.

#### Root causes identifiées
1. **Ollama 0.18.0 outdated** : `think: false` ignoré → le modèle passe tous les tokens en raisonnement interne
2. **Multi-image** : envoyer 3 images d'un coup faisait crasher ou timeout
3. **`normalizeAnalysis`** : cherchait `extractedInfo` mais qwen3-vl retourne une structure plate
4. **`analysisProvider` enum** : valeur custom non acceptée par le modèle Mongoose

#### Fixes appliqués

**Serveur Ollama (192.168.110.103) :**
- Mise à jour Ollama **0.18.0 → 0.20.3** (fix du `think: false`)
- Installation de **deepseek-ocr** (modèle OCR 3.3B, ultra-rapide)
- **OLLAMA_NUM_PARALLEL=3** dans l'override systemd (parallélisme OCR)

**Backend (192.168.110.59, PM2 `ai-builder-v2`) :**

`src/services/vision.service.js` — Pipeline OCR entièrement réécrit :
- **Étape 1** : `deepseek-ocr` via `/api/generate` (~2-5s/image, batches de 3 en parallèle)
- **Étape 2** : `qwen3.5:9b` pour structurer tout le texte OCR en JSON (~22s)
- Fallback sur Claude Vision si le pipeline local échoue
- Headers `Content-Type: application/json` explicites (fix du `think: false` via axios)

`src/services/migration.service.js` :
- `normalizeAnalysis` corrigé : gère les structures plates ET wrappées (`extractedInfo` ou direct)
- `maxPages: 8` (au lieu de 3)
- Progress granulaire (5% → 15% → 100%)
- Callback `onScreenshot` pour sauvegarde streaming

`src/services/screenshot.service.js` :
- Délais réduits : `setTimeout(1500)` → `500ms`, `setTimeout(800)` → `300ms`
- Scroll plus rapide : `distance: 600`, `maxScrolls: 10`, `interval: 80ms`
- Support callback `onScreenshot` pour pipeline streaming
- Dédoublonnage des liens nav (`Set`)
- Capture jusqu'à 7 pages additionnelles
